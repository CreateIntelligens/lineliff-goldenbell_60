1. 金鐘60星光打Call活動企劃
專案概述
活動名稱 金鐘60星光打Call
專案目標 透過LINE官方帳號，為心愛的金鐘節目與藝人加油打氣，並獲得專屬紀念海報，提升用戶參與度並延長曝光週期。
執行平台
* 主要平台：LINE官方帳號（Fanpokka）
* 技術架構：LIFF（LINE Front-end Framework）
* 輸出格式：個人化圖片海報
使用者流程設計
完整用戶旅程
1. 進入LIFF頁面 → 點擊推廣出去的URL，提供LIFF頁面連結進行個人化製作
2. 加入LINE官方帳號 → 首先，加入Fanpokka的絲路通行證的LINE官方帳號
3. 輸入應援語句 → 在LINE聊天室中輸入「我要為______加油！」或「給金鐘60的一句話」，表達您的支持
4. 生成金鐘海報 → 系統將自動生成一張獨一無二的應援海報，海報模板採用金鐘60主視覺設計，並包含官方LOGO
5. 貼上金鐘60標籤 → 作為得獎感言受眾
6. 【新增】分享邀請功能 → 點擊「邀請朋友一起打Call」按鈕，生成帶有OG內容的分享URL，邀請更多朋友參與活動
7. 【新增】下載下載功能 → 點擊「下載到官方帳號」按鈕，將生成圖片下載並下載至Fanpokka官方帳號
8. 分享到社群 → 輕鬆分享您的專屬海報到個人社群平台（Line FB）
功能規格說明
核心功能流程
1. 應援輸入機制
   * 輸入格式：
      * 格式1：「我要為______加油！」
      * 格式2：「給金鐘60的一句話」
   * 字數限制：建議30-50字以內
   * 內容驗證：基本不當內容過濾
2. 海報生成系統
   * 即時個人化：每張海報都是獨一無二，增加分享動機
   * 視覺一致性：配合金鐘60主視覺，整體宣傳統一
   * 內容整合：包含官方LOGO與頒獎典禮資訊
   * 用戶內容：融入用戶的應援文字
3. 【新增】分享邀請系統
   * OG標籤設定：固定OG內容，不包含個人作品預覽
   * 邀請連結：生成專屬邀請URL，包含分享者資訊
   * 病毒傳播：被邀請者透過固定邀請內容進入活動並參與製作
   * OG內容規範：
      * 標題：「金鐘60星光打Call｜為心愛的節目瘋狂應援！」
      * 描述：「金鐘盛典即將登場！快來製作你的專屬應援海報，為最愛的節目和藝人加油打氣，一起點亮金鐘星光大道！」
      * 分享圖規格：
         * FB格式：1200x630px (1.91:1橫式比例)
         * LINE格式：1200x1200px (1:1正方形比例)
         * 設計要求：主要視覺元素需置中，確保在兩種裁切比例下都能完整顯示
         * 安全區域：重要文字和LOGO需在中央600x600px安全區域內
         * 內容：金鐘60主視覺 + 活動標題 + 邀請文案
4. 【新增】官方下載系統
   * 作品下載：用戶可將作品下載下載至「個人作品褲」
   * 個人作品庫：點擊展開所有生成過的內容
   * 重複下載：支援多次下載和管理個人作品
5. 限制機制
   * LINE收會員：Fanpokka收官方帳號會員，需加入好友才可以使用相關功能
   * 數據收集：Fanpokka收數據，網站累積應援內容

2. 我的得獎感言活動企劃
專案概述
活動名稱 我的得獎感言
專案目標 金鐘獎典禮結束後，Fanpokka 透過 LINE 官方帳號發起「如果我是得獎者」活動，邀請粉絲化身金鐘得主，寫下屬於自己的「獲獎感言」。此舉既能延續典禮熱度，也能鼓勵粉絲主動創造內容並分享到社群，形成二次話題擴散。
執行平台
* 主要平台：LINE官方帳號（Fanpokka）
* 技術架構：LIFF（LINE Front-end Framework）
* 輸出格式：個人化金鐘感言卡
使用者流程設計
完整用戶旅程
1. 加入LINE官方帳號 → 成為Fanpokka LINE官方帳號會員
2. 接收活動推播 → 收到「如果你是金鐘獎得主，你想說的得獎感言是什麼？」訊息
3. 輸入感言文字 → 在LINE中直接輸入並上傳您的獲獎感言
4. 生成感言卡 → 系統自動生成個人化「金鐘感言卡」（搭配金鐘60主視覺）
5. 【新增】分享邀請功能 → 點擊「邀請朋友分享感言」按鈕，生成帶有OG內容的分享URL，邀請更多朋友參與活動
6. 【新增】下載下載功能 → 點擊「下載到官方帳號」按鈕，將感言卡下載並下載至Fanpokka官方帳號
7. 一鍵分享社群 → 將您的感言卡輕鬆分享到個人社群平台（LINE、Facebook）
8. 停留結果頁面 → 生成後會留在生成結果頁面，可重複分享和下載
功能規格說明
核心功能流程
1. 感言輸入機制
